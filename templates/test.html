Forschreitender Verlauf des Kurses

Im weiteren Verlauf des Kurses wurde ein Webserver mit Flask gestartet. Dieser läuft erstmal nur lokal und kann dafür genutzt werden unsere html-Dateien darzustellen. Mitlerweile wurden viele unter Dateien erstellt und es ist wichtig, dass der Überblick darüber erhalten bleibt.
Unter "templates" liegen alle html-Dateien. mit render_template("Dateiname") geht der Flask automatisch in den templates Ordner und kann die angegebene html-Datei einlesen und darstellen.
Unter "layouts" in "templates" liegt die main.html Datei. In ihr wird das Grundgerüst der Website beschrieben. Alle nötigen css-Dateien werden eingelesen und der gesamte Inhalt wird bereits mit der skeleton klasse "container" gestyled. Die anderen html Dateien erben (inheriten) von der main datei über die Schreibweise von "jinja".

Über die URLs können auch Parameter übergeben werden. Wenn beispielsweise ein ?name=irgendein_Name in der URL steht, kann dieser abgefragt werden und weiter verarbeitet werden Die soll hier geschehen. Nach diesem Satz wird der in der URL definierte Name angezeigt werden:
{% if name!= None %} {{name}} {% endif %}

Falls die Variablen nicht definiert sind wird Python ein "None" an die Stelle setzen. Dies kann jedoch umgangen werden indem man eine if Bedingung abfragt:
(In Python:) if name != None: tue etwas
(In den html-files) {} % if name!= None % der code muss in die geschweifte Klammer (ist hier nicht, damit html es nicht direkt als klammer erkennt) und muss wieder geschlossenen werden über {} % endif %
